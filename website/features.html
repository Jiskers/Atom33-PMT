boards

A corkboard that never runs out of room.

Pin sticky notes, reference palettes, and mechanic cards anywhere on a zoomable, resizable canvas — pan, zoom, and pinch just like a real board. Lock a module in place once it's final, rotate it for that pinned-up feel, or straighten it back out in one click.

  • Four canvas sizes, from a tight jam board to a huge planning wall
  • Per-module settings — swatch colors, hide-completed toggles, status pills
  • Everything autosaves and survives a reload, mid-drag

game-dev module pack

Built for the shape of an actual game project.

Beyond sticky notes and checklists, devboard ships modules that already know what a game dev needs to track — a design doc outline, an asset pipeline, bugs with severity, and a dated playtest log.

  • GDD outline — heading + notes sections you add on the fly
  • Asset pipeline — cycle each asset through concept → wip → final → in-engine
  • Bug report — severity pill, repro steps, open/fixed toggle
  • Playtest log — auto-dated running notes from every session

spreadsheets

A real formula engine, not a checklist pretending to be one.

Cell references, ranges, SUM(), and the standard arithmetic operators — enough for a scope budget or a loot table without opening a whole other program. Resizable well past the original fixed grid.

  • References update live as you edit source cells
  • Grows up to 26×200 cells per sheet
  • Formulas evaluate safely — no arbitrary code execution

code + run

Write it, run it, right there.

A code file with syntax-aware editing for HTML/CSS/JS/PHP, and a real Run button that opens a browser-chrome preview tab — good for devlog pages, small prototypes, or just testing a snippet.

extensibility

The built-ins use nothing a plugin couldn't.

Every module, view, and dashboard widget in devboard registers through the same three open contracts. There's no special core-only shortcut — a community add-on plays on exactly the same field.

registerModule

Pinnable board content — a sticky note, a bug report, anything with its own data and a card body.

registerView

A whole new file type in the tree — board, kanban, sheet, draw, and code all register exactly this way.

registerWidget

A Home dashboard tile — gets read access to the whole project, not just its own data, unlike a module.

Multiple projects. Full undo history. Real migrations.

Switch between as many local projects as you want. Every edit is undoable. And when a module or view's data shape changes between versions, devboard migrates old saves forward automatically — nothing left behind on update.
