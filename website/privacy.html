Privacy Policy

Last updated: July 2026
Not legal advice. This is a plain-language template describing what devboard actually does today, written for a local-first app with no account system and no analytics. It has not been reviewed by a lawyer. If devboard adds accounts, cloud sync, or analytics, this policy needs a real rewrite (and likely a compliance review) before that ships — see section 6.

1. The short version

devboard doesn't collect your project data, doesn't require an account, and doesn't run analytics or telemetry. Everything you create — boards, notes, kanban cards, sheets, drawings, code — stays on your own device.

2. What we don't collect

  • We don't collect your name, email, or any personal information — devboard doesn't ask for any.
  • We don't collect or transmit your project content. It never leaves your device.
  • We don't run analytics, ad tracking, or usage telemetry in the desktop app.

3. Where your data actually lives

The desktop app stores your projects as local files in your system's application data folder. The web build (if you're using it in a browser instead of the desktop app) stores projects in your browser's local storage. In both cases, that data is under your control — uninstalling the app or clearing that storage removes it.

4. Network requests the app does make

The desktop app makes exactly one kind of outbound request on its own: it periodically checks a public GitHub endpoint to see if a newer version is available. This request doesn't include your project content, an account identifier, or anything else about you beyond what any HTTP request inherently carries (like your IP address, visible to GitHub as the host — not to us).

5. This website

This marketing site loads fonts from Google Fonts and, on the Releases page, fetches public release data from the GitHub API in your browser. Both are third-party requests made directly by your browser to Google and GitHub respectively, subject to their own privacy policies — this site itself doesn't run analytics or set tracking cookies.

6. Future account-based features

We plan to eventually offer optional sign-in (for example, via Google) enabling features like cloud sync. Those features will be opt-in and off by default, and this policy will be rewritten — with a clear explanation of exactly what data is involved — before any such feature ships. Nothing described in this section exists in the current version.

7. Children's privacy

devboard is not directed at children under 13 and we don't knowingly collect information from them — in practice, since we don't collect personal information from anyone, this is largely moot for the current version of the app.

8. Your control over your data

Because your data lives entirely on your own device, you have complete control over it — back it up, move it, or delete it by managing your local files or browser storage directly. There's no account for us to delete on our end, because there's no account.

9. Changes to this policy

We'll update the "Last updated" date above whenever this policy changes, and we'll flag any material change (especially anything involving new data collection) prominently rather than silently.

10. Contact

Questions about this policy can be raised as an issue on the GitHub repository.
